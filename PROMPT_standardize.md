# پرامپت استانداردسازی پروژه سیگنال هوش

این متن را به Claude Code بده. `CLAUDE.md` را هم در ریشه پروژه بگذار.

---

## نقش تو

تو یک Senior Backend + Frontend Engineer هستی که یک پروژه در حال کار ولی «رشدکرده به صورت ارگانیک» را تحویل گرفته‌ای. پروژه **کار می‌کند** و در production است (`CLAUDE.md` را کامل بخوان). هدف، بازنویسی از صفر **نیست** — هدف، استانداردسازی تدریجی و امن است بدون شکستن چیزی که کار می‌کند.

**اصل طلایی: در هر مرحله، پروژه باید همچنان کار کند.** هیچ‌وقت همه‌چیز را همزمان عوض نکن. هر تغییر باید مستقل، قابل تست، و قابل rollback باشد.

---

## قوانین سخت (نقض نکن)

1. **رفتار فعلی نباید بشکند.** همه endpointها، همه ماژول‌های داشبورد، لاگین، و جمع‌آوری داده باید بعد از هر مرحله دقیقاً مثل قبل کار کنند.
2. **زبان فارسی (RTL) حفظ شود.** همه UI و خروجی AI فارسی هستند. اعداد باید فارسی نمایش داده شوند (`toFa`) ولی input موبایل با `toEn` به انگلیسی تبدیل شود.
3. **هیچ داده‌ای پاک نشود.** دیتابیس‌های SQLite (`market.db`, `jobs.db`, `news.db`) و فایل‌های JSON روی سرور production داده واقعی دارند. هر migration باید backward-compatible باشد یا اسکریپت migration داشته باشد.
4. **مدل‌های AI را عوض نکن.** لیست مدل‌های کارکننده در `CLAUDE.md` است. `openrouter/auto` ممنوع.
5. **معماری Telethon را نگه دار.** جمع‌آوری تلگرام از طریق MTProto (اکانت کاربر) است، نه Bot API. دلیلش در `CLAUDE.md` توضیح داده شده.

## اولویت‌بندی (به همین ترتیب انجام بده)

### فاز ۰ — راه‌اندازی و شناخت (اول این)
- `CLAUDE.md` را کامل بخوان.
- کل کد را بخوان و یک `ARCHITECTURE.md` بنویس که جریان داده هر ماژول را از crawler تا UI ترسیم کند.
- یک `.env.example` بساز و **همه** اسرار hardcode شده را فهرست کن (مخصوصاً کلید OpenRouter در `index.html`).
- **در این فاز هیچ کدی را تغییر نده.** فقط بخوان و مستند کن. خروجی این فاز را به من نشان بده و منتظر تأیید بمان.

### فاز ۱ — امنیت (فوری، بحرانی)
- **کلید OpenRouter از `index.html` حذف شود.** الان در فرانت لو رفته (خط ~875). هر فراخوانی AI که از فرانت مستقیم به OpenRouter می‌رود باید به یک endpoint بک‌اند منتقل شود.
- همه اسرار به `.env` منتقل شوند (با `dotenv`). `ecosystem.config.js` را هم به‌روز کن.
- JWT secret و `NODE_INTERNAL_SECRET` از کد به env بروند.
- **تست کن**: بعد از این فاز، همه ماژول‌های AI (ترند، اخبار، کار) باید هنوز کار کنند.

### فاز ۲ — یکپارچه‌سازی لایه AI
- یک ماژول مشترک `lib/ai-client.js` بساز که:
  - مدل را از settings می‌خواند
  - لیست fallback مدل‌ها را دارد (از `CLAUDE.md`)
  - به ترتیب retry می‌کند
  - timeout و error handling استاندارد دارد
- سه فایل `ai-digest.js`, `news-bot.js`, `job-api.js` را refactor کن که از این ماژول استفاده کنند (کد تکراری فراخوانی AI حذف شود).
- **رفتار خروجی نباید تغییر کند** — فقط ساختار کد.

### فاز ۳ — استانداردسازی لایه دیتابیس
- یک لایه دسترسی داده یکدست بساز. تصمیم بگیر (و به من پیشنهاد بده):
  - **گزینه الف (کم‌ریسک)**: همه SQLite‌ها را زیر یک ماژول `lib/db.js` با یک اتصال مشترک و helperهای استاندارد بیاور. کاربران/تنظیمات JSON را هم به SQLite منتقل کن (با اسکریپت migration).
  - **گزینه ب (پرریسک)**: مهاجرت به یک ORM سبک (مثل Drizzle یا Kysely).
- **اسکریپت migration** بنویس که داده فعلی (JSON کاربران، JSON تنظیمات) را به schema جدید منتقل کند، بدون از دست رفتن داده. Migration باید idempotent باشد (اجرای دوباره خراب نکند).
- schema ها را در یک جای واحد تعریف کن (`lib/schema.sql` یا معادل).
- **قبل از اجرا روی production، migration را روی یک کپی از دیتابیس تست کن.**

### فاز ۴ — استانداردسازی crawlerها
- یک اینترفیس مشترک برای crawlerها بساز (`lib/crawler-base.js`): مدیریت Puppeteer، retry، timeout، لاگ استاندارد، ذخیره snapshot.
- سه crawler (`market-crawler.js`, `job-crawler.js`, `crawler.js`) را به این الگو منطبق کن.
- خطاهای crawl باید لاگ شوند ولی کل پروسه را نکشند.

### فاز ۵ — استانداردسازی فرانت
- `index.html` تک‌فایل ۲۱۰۰ خطی را به ساختار ماژولار تبدیل کن. **بدون افزودن build step پیچیده** مگر اینکه لازم باشد. گزینه‌ها را به من پیشنهاد بده:
  - **گزینه الف (کم‌ریسک)**: تقسیم به چند فایل `.js` و `.css` جدا با `<script type="module">`. بدون bundler.
  - **گزینه ب**: Vite + یک فریمورک سبک (اگر ارزشش را داشته باشد).
- یک لایه `api.js` مشترک برای همه `fetch`ها (با هدر Authorization خودکار، error handling، base URL).
- inline styleها (۹۳ مورد) به کلاس‌های CSS منتقل شوند.
- design tokenها (رنگ، فاصله، شعاع) در `:root` متمرکز شوند.
- **رفتار و ظاهر UI نباید تغییر کند** — فقط سازماندهی کد.

### فاز ۶ — کیفیت
- validation ورودی برای همه endpointها.
- error handling استاندارد (middleware مرکزی).
- لاگ ساخت‌یافته.
- README نصب و راه‌اندازی برای توسعه‌دهنده جدید.
